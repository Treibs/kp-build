## The plain-terms story

Ask an AI for a Sui contract today and it may well write last year's Sui. The counter-contract proof: an unaided model writing a plain counter contract from parametric memory produced pre-2024-edition Move — `struct Counter` — which fails to compile on sui mainnet-v1.74.1 (visibility annotations are mandatory in the 2024 edition). One mechanical fix (`public struct`) and the same contract builds green. The pack is a receipt-backed cheat sheet: every rule already ran through the real compiler, and every rule ships with the broken form the model would otherwise write.

## The RED/GREEN two-sided gate

Each rule carries a GREEN fixture (must compile); 32 of 39 rules also carry a RED fixture (must FAIL with a pinned error fragment in `expected_error.txt`). The refresh mechanic: Sui ships a new mainnet toolchain every ~2 weeks (the full re-verify loop is specified in [CONTEXT.md](CONTEXT.md)). This is the first pack whose staleness check is mechanical in both directions — with one disclosed exception: the test-scenario beat's `#[test]` body is not compiled by the plain-build gate (see [claims/test-scenario-green.md](claims/test-scenario-green.md)).

Pack grew 47 → 61 claims. Same honesty invariant as the revision beat: experiment 1/2 numbers were measured on the pre-deepening pack. The deepening's held-out effect was then measured by pre-registered experiment 3 (table above): deepened pack 4/6 vs pre-deepening pack 2/6 compile-pass for claude-haiku-4-5, with the taught `use-self` corner recurring only in the pre-deepening arm.

**Payload-salience experiment (2026-07-08).** The salience fork was taken: same 98 claims, same model, six fresh held-out tasks — only the *form* of the loaded payload varied (protocol and pre-registration in `docs/experiments/sui-payload-salience/`, frozen at `c170058` before any answer). Four arms: base, **full** (the standard CONTEXT.md + sorted-claims payload used by every prior experiment, 49,656 B), **slim** (the same sorted claim statements once, briefing stripped — mechanical redundancy removal, 24,639 B), **sheet** (35 imperative rule lines derived 1:1 from the claims, 5,887 B). Result: **rule branch 1 — payload form matters.** Sheet compile-passed 3/6 vs full 1/6, meeting the pre-registered ≥ +2 bar exactly; slim scored 0/6, so the win is the sheet's imperative form and/or compression (a disclosed confound), NOT redundancy removal — mechanically slimming the standard payload did not help. The sheet was adopted as the recommended answer-time load format (**since withdrawn** — see the confirmation draw below); the full payload remained the archival/verification form throughout. Qualifiers recorded with the same prominence: the bar was met with zero margin at n=6 under two disclosed comparisons; the sheet's three passes all carry warnings (clean-compile 0/6 vs full 1/6 — the sheet buys compile-pass, not cleanliness); this was the hardest draw of the series (base 0/6), so absolute numbers are not comparable across experiments; and the pre-registered mechanism analysis did NOT cleanly support the salience hypothesis — the sheet's three failing rows still ignored three loaded rules (two with exact pinned messages — `struct-field-store`, `implicit-field-copy` — plus the create-currency call shape at green-claim scope). The ignored-while-loaded phenomenon now stands across three consecutive experiments and every payload form tested. A confirmation draw (sheet vs full only, one comparison) is recorded as required before any emit-sheet tooling is built.

**Sheet-vs-full confirmation draw (2026-07-09) — the sheet adoption is WITHDRAWN.** The required confirmation draw was run: six fresh held-out tasks, one pre-registered comparison (sheet vs full, payloads reused byte-identical by SHA-256 pin), frozen at `5e605c9` before any answer (protocol in `docs/experiments/sui-sheet-confirm/`). Result: **rule branch 3** — full compile-passed **4/6** against the sheet's **1/6** (base anchor 0/6, the same difficulty extreme as the salience draw). Per the frozen rule the salience adoption is withdrawn: **the full payload is again the recommended answer-time load format**, the sheet reverts to an experimental view of the pack, and `emit-sheet` tooling stays unbuilt. Pooled across both draws (observational): sheet 4/12 vs full 5/12 — no form signal survives; the salience win, which met its bar with zero margin, reads as draw noise. The withdrawal is reported with the same prominence as the adoption, per the rule frozen before the data. Mechanism analysis (observational, strict recurrence standard): the ignored-while-loaded phenomenon stands in a **fourth** consecutive experiment and in both forms — the full arm's two failing rows both carry the loaded `use-self` rule (exact pinned message), and the sheet's five failing rows carry four exact-message loaded-rule-ignored events (`module-address-form`, `param-mut`, `block-statement-semicolon` ×2). New untaught classes recorded for the round-4 seed ledger: Rust compound assignment `+=` (E01002, 2 rows/3 sites), the E07001 referential-transparency borrow shape, a uid-vs-ID event field, and base's hallucinated test-only `coin::burn_for_testing`. Form manipulation has now been tried and does not solve rule application.

**Deepening round 4 (2026-07-09).** A fourth gap-seeded probe (15 fresh tasks across five territories seeded from the enriched post-experiment ledger — SUI-import, Rust syntax bleed, Rust API bleed, generic-transfer-key-bound, object identity — dual model, 98-claim pack loaded byte-identical to the confirmation draw's full payload, pinned oracle; protocol and ledger in `docs/deepening/round-4/`) surfaced 4 compile failures for claude-haiku-4-5 and 1 for claude-sonnet-4-6, carrying seven distinct root causes of which four were beat-worthy. Four fixture-proven beats: `borrow-arg-alias` (E07001 — a second borrow of a field in the argument list of a call already holding `&mut` on that field; the sheet-confirm class, promoted on recurrence), `compound-assignment` (E01002 — Rust's `+=` does not parse; ×3 cumulative answers), `while-condition-parens` (E01002 — Move 2024 parenthesizes `while` conditions; a new parse-level sibling elicited by the Rust-syntax territory), and `std-mem-swap` (E03002 — there is no `std::mem`; the Move idiom is an `Option` slot with `std::option::swap`, ×3 cumulative for the family). Not taught: the round-3 `param-mut` rule ignored while loaded (×2 in one answer, recorded), and two new single-answer classes deferred by the standing rule (`vector-destroy-empty` E06001, `vector-literal-annotation` E04010). Three territories were recorded clean with their corners demonstrably exercised: all six SUI-import answers wrote `use sui::sui::SUI` (the ×4-cumulative top ledger class did not reproduce under targeted elicitation), all six generic-bound choices were sound (second consecutive clean probe for that class), and all six object-identity answers passed with the identity corners engaged (`object::delete`, `ID`-typed event fields; one answer keyed its registry by the object's `address` form — semantically defensible, noted in the triage). Pack grew 98 → 110 claims. Same honesty invariant: every experiment number above was measured on the pack size recorded for it; round 4's held-out effect is unmeasured until the next pre-registered tier-2 falsification.

## What this pack does NOT do
