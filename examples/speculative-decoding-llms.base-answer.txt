# Related Work

**Speculative decoding** accelerates autoregressive LLM inference by having a cheap draft model propose tokens that a target model verifies in parallel, accepting the longest correct prefix; the foundational formulations are speculative sampling (arXiv:2302.01318, Chen et al.) and speculative decoding (arXiv:2211.17192, Leviathan et al.), both of which preserve the target distribution exactly.

**Self-drafting and parallel-head methods.** Medusa (arXiv:2401.10774) removes the separate draft model by attaching multiple parallel prediction heads to the target model, each forecasting a future position, and verifying candidate continuations with a tree-attention mask. Independently, Jacobi/Lookahead decoding (Lookahead Decoding, arXiv:2402.02057) reframes generation as solving a nonlinear system via Jacobi iteration, producing n-grams in parallel without any draft model, trading guaranteed acceptance for draft-free parallelism. These approaches reduce draft latency but suffer lower acceptance because parallel heads predict positions independently rather than autoregressively.

**Feature-level autoregressive drafting.** EAGLE (arXiv:2401.15077) showed that drafting at the *feature* (second-to-top hidden state) level, rather than the token level, is more regular and predictable; it runs a lightweight autoregressive head over features plus the shifted token sequence, substantially raising acceptance. EAGLE-2 (arXiv:2406.16858) made the draft tree *dynamic and context-aware*, using the draft model's confidence to expand and prune the tree per-context instead of using a fixed tree, improving the acceptance-vs-overhead trade-off. EAGLE-3 (arXiv:2503.01840) dropped the feature-reconstruction loss and fused *multiple layers* of the target model ("training-time test"), scaling draft quality with data and further boosting speedup.

**Tree verification.** SpecInfer (arXiv:2305.09781) introduced tree-based, multi-candidate speculative inference, merging many small draft sequences into a token tree and verifying all candidates simultaneously with tree attention, decoupling acceptance gains from sequential verification cost.

**Regime-dependent limitations.** Recent work documents that speculative speedups erode under high-throughput serving: when batches are large, the target forward is already compute-bound, so verifying speculative tokens competes for the same FLOPs and net speedup collapses or inverts (analysis in arXiv:2406.14066, on continuous-batching serving). Long-context reasoning workloads similarly degrade gains as KV-cache and verification overheads grow, motivating batch- and context-adaptive speculation policies — the gap our method targets.
