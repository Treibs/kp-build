Should agent memory be parametric (stored in/near model weights, self-updating) or contextual (external, editable, retrieved)?

### Parametric / self-updatable memory
Integrate a fixed-size self-updatable memory pool into the transformer's latent space so the model absorbs new knowledge directly and retains it across ~1M updates without retrieval latency or degradation.

*Held by:* [[papers/wang2024memoryllm]]

### Contextual / external memory (non-parametric continual learning)
Keep memory as explicit, editable external data (notes, KGs, temporal graphs) — auditable, updatable, and supporting non-parametric continual learning that better mimics the dynamic, interconnected nature of human long-term memory.

*Held by:* [[papers/gutierrez2025hipporag2]], [[papers/rasmussen2025zep]], [[papers/xu2025amem]], [[papers/chhikara2025mem0]]

### Both are needed; the open question is allocation
Memory should be taxonomized into parametric and contextual forms with operations spanning both; the unresolved problem is which knowledge belongs where and how to keep multi-source memory consistent.

*Held by:* [[papers/du2025rethinkingmemory]], [[papers/huang2026secondhalf]]
