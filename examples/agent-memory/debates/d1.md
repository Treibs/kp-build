When should an agent use long-context LLMs versus retrieval (RAG) versus an explicit external memory store?

### Long-context wins on accuracy when resourced
When sufficiently resourced, long-context LLMs consistently outperform RAG on average and on QA (especially Wikipedia-based); the right move is to feed more context when you can afford it, optionally routing cheaper queries to RAG.

*Held by:* [[papers/li2024ragvslongcontext]], [[papers/li2025longcontextvsrag]]

### Retrieval/explicit memory wins on cost and some query types, and long context is unreliable
RAG is far cheaper and wins on dialogue/general queries; long-context models suffer 'lost-in-the-middle' positional failures and a ~30% accuracy drop on sustained interactions, so explicit retrieval/memory is preferable for long-horizon, cost-sensitive deployment.

*Held by:* [[papers/liu2023lostmiddle]], [[papers/li2025longcontextvsrag]], [[papers/wu2024longmemeval]], [[papers/chhikara2025mem0]]

### Hybrid routing is the practical resolution
Neither is universally best; route each query to RAG or long-context via model self-reflection (Self-Route) to capture most of the accuracy at a fraction of the cost.

*Held by:* [[papers/li2024ragvslongcontext]]
