**Scope:** Persistent and long-term memory for autonomous LLM agents from 2023 to 2026. Covers: (a) foundational architectures (OS-style virtual context, memory streams, episodic buffers); (b) the episodic-vs-semantic memory distinction and how each is instantiated; (c) memory consolidation, updating, and forgetting (Ebbinghaus curves, ADD/UPDATE/DELETE operators, parametric self-update); (d) retrieval mechanisms (vector RAG, knowledge-graph / hippocampal indexing, temporal KGs); (e) the long-context-vs-RAG-vs-explicit-memory debate (cost vs accuracy, routing); and (f) evaluation/benchmarks for long-term interactive memory. ## Open problems (where new work goes)

- **No current memory agent masters all four core memory competencies (accurate retrieval, test-time learning, long-range understanding, selective forgetting) simultaneously.** (open) — If a system can retrieve but cannot learn at test time or forget stale facts, it fails in real deployments. The benchmark result shows the field has optimized retrieval in isolation while leaving consolidation, continual learning, and forgetting unsolved — these are the capabilities that distinguish memory from search. *Flagged by [hu2025memoryagentbench].*
- **Long-term conversational memory remains substantially below human performance on long-range temporal and causal reasoning, even with long-context LLMs or RAG.** (open) — Temporal reasoning (what was true when, knowledge updates, abstention) is precisely what persistent agents need over months of interaction. The ~30% accuracy drop on LongMemEval and the human gap on LoCoMo show neither stuffing context nor naive retrieval solves it; temporal modeling of memory is an unsolved core problem. *Flagged by [maharana2024locomo], [wu2024longmemeval].*
- **Adding knowledge-graph structure to retrieval improves sense-making and associativity but degrades basic factual memory below plain vector RAG — a tension no single architecture had resolved before HippoRAG 2, and the general trade-off remains active.** (partially-addressed) — Structured memory (KGs, temporal graphs) is the leading direction for associative/multi-hop recall, but if it sacrifices simple fact lookup it is not a drop-in replacement. Reconciling structured and unstructured memory so both factual and associative recall improve is a live design problem. *Flagged by [gutierrez2025hipporag2], [rasmussen2025zep].*
- **It is unresolved whether memory should be parametric (self-updating weights / latent pools) or contextual (external editable stores), and how to combine them; multi-source memory integration is flagged as an open research topic.** (open) — Parametric memory (MEMORYLLM) avoids retrieval latency and integrates knowledge but is opaque and hard to audit/delete; contextual memory is editable and inspectable but bolted-on. The field lacks a principled account of which knowledge belongs where and how to keep them consistent — directly bearing on auditability, privacy, and forgetting. *Flagged by [du2025rethinkingmemory], [wang2024memoryllm], [pink2025episodic].*
- **Episodic memory — single-shot learning of instance-specific, contextually-bound experiences — is argued to be a missing capability distinct from semantic memory and RAG, and no system yet supports all five proposed episodic-memory properties together.** (open) — Most deployed 'memory' is semantic (facts about the user) or flat retrieval. True episodic memory (explicit reasoning over instance-specific events with contextual binding) is what lets agents learn from a single experience without retraining — the position paper frames this as the catalytic missing piece for long-term agents. *Flagged by [pink2025episodic].*
- **Selective forgetting / memory consolidation lacks principled, evaluated mechanisms; existing approaches range from Ebbinghaus-style decay to LLM-driven ADD/UPDATE/DELETE operators, but no consensus on when and what to forget.** (open) — Unbounded memory growth causes cost explosion and retrieval noise; wrong forgetting loses critical facts or fails to update superseded ones. Forgetting and consolidation are listed as core operations yet are the least benchmarked, making this a high-value gap for deployable agents. *Flagged by [zhong2023memorybank], [chhikara2025mem0], [du2025rethinkingmemory], [hu2025memoryagentbench].*
- **Memory evaluation itself is immature: prior benchmarks are limited in memory levels, interactive scenarios, and metric diversity, and most measure single axes rather than effectiveness, efficiency, and capacity jointly.** (open) — Without benchmarks that test participation vs. observation, factual vs. reflective memory, and cost/capacity alongside accuracy, reported gains may not transfer to real long-horizon use. The proliferation of new benchmarks (LoCoMo, LongMemEval, MemBench, MemoryAgentBench) signals the community has not converged on how to measure memory. *Flagged by [tan2025membench], [hu2025memoryagentbench], [zhang2024memorysurvey].*
- **The long-context-vs-RAG-vs-explicit-memory choice is not settled: results depend on resourcing, question type, and dataset, and dynamic routing (e.g., Self-Route) is a heuristic rather than a principled solution.** (open) — Practitioners must choose between paying for long context, building retrieval, or maintaining explicit memory stores. Because long-context still has 'lost-in-the-middle' failures and RAG wins on cost and on some query types, there is no general rule; a principled cost-aware policy for when to use each remains open. *Flagged by [li2024ragvslongcontext], [li2025longcontextvsrag], [liu2023lostmiddle].*

## Open debates / contested points

- **When should an agent use long-context LLMs versus retrieval (RAG) versus an explicit external memory store?**
    - *Long-context wins on accuracy when resourced* ([li2024ragvslongcontext], [li2025longcontextvsrag]): When sufficiently resourced, long-context LLMs consistently outperform RAG on average and on QA (especially Wikipedia-based); the right move is to feed more context when you can afford it, optionally routing cheaper queries to RAG.
    - *Retrieval/explicit memory wins on cost and some query types, and long context is unreliable* ([liu2023lostmiddle], [li2025longcontextvsrag], [wu2024longmemeval], [chhikara2025mem0]): RAG is far cheaper and wins on dialogue/general queries; long-context models suffer 'lost-in-the-middle' positional failures and a ~30% accuracy drop on sustained interactions, so explicit retrieval/memory is preferable for long-horizon, cost-sensitive deployment.
    - *Hybrid routing is the practical resolution* ([li2024ragvslongcontext]): Neither is universally best; route each query to RAG or long-context via model self-reflection (Self-Route) to capture most of the accuracy at a fraction of the cost.
- **Should agent memory be parametric (stored in/near model weights, self-updating) or contextual (external, editable, retrieved)?**
    - *Parametric / self-updatable memory* ([wang2024memoryllm]): Integrate a fixed-size self-updatable memory pool into the transformer's latent space so the model absorbs new knowledge directly and retains it across ~1M updates without retrieval latency or degradation.
    - *Contextual / external memory (non-parametric continual learning)* ([gutierrez2025hipporag2], [rasmussen2025zep], [xu2025amem], [chhikara2025mem0]): Keep memory as explicit, editable external data (notes, KGs, temporal graphs) — auditable, updatable, and supporting non-parametric continual learning that better mimics the dynamic, interconnected nature of human long-term memory.
    - *Both are needed; the open question is allocation* ([du2025rethinkingmemory], [huang2026secondhalf]): Memory should be taxonomized into parametric and contextual forms with operations spanning both; the unresolved problem is which knowledge belongs where and how to keep multi-source memory consistent.
- **Is episodic memory a distinct, missing capability, or is semantic memory / retrieval sufficient for long-term agents?**
    - *Episodic memory is the missing piece* ([pink2025episodic]): Single-shot, instance-specific, contextually-bound episodic memory is qualitatively different from semantic memory and RAG and is the catalytic capability for long-term agents; the field should focus explicitly on its five required properties.
    - *Reflective/experiential episodic buffers already deliver much of the value* ([shinn2023reflexion], [park2023generative]): Storing and reflecting on past episodes in an episodic buffer (Reflexion) or memory stream with reflection (Generative Agents) already improves decision-making and believability without weight updates, suggesting practical episodic mechanisms are tractable today.
    - *Both episodic and semantic are facets of a broader memory taxonomy* ([huang2026secondhalf], [du2025rethinkingmemory]): Episodic and semantic are two of several cognitive mechanisms (alongside sensory, working, procedural); the design question is how to instantiate and coordinate all of them rather than privileging one.
- **Does adding structure (knowledge graphs, temporal graphs) to memory help or hurt compared to plain vector retrieval?**
    - *Structure helps multi-hop, associative, and temporal recall* ([gutierrez2024hipporag], [rasmussen2025zep], [gutierrez2025hipporag2]): KG/PageRank indexing enables single-step multi-hop retrieval with large accuracy and cost wins; temporal KGs add auditable temporal reasoning and beat MemGPT and baselines.
    - *Structure can hurt basic factual recall* ([gutierrez2025hipporag2]): Graph-augmented RAG improves sense-making and associativity but historically dropped basic factual memory below standard vector RAG — a regression that had to be explicitly fixed.
