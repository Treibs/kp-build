What we get, what it costs, and when it pays

kp-build works where it's meant to and is provably worthless where it isn't — that gap is the whole decision. On a model-weak topic it lifted a downstream agent from f1 0.37 → 0.91 and cut citation fabrication from 37.5% → 0%. The product is sound; the unfunded gap is a cheap pre-flight check so we never pay to build a package that adds nothing.

Cost-benefit · ROI

Build once; every reuse is near-free

The model is amortization. Build a package once (~0.6–1.5M tokens); each consumer then loads a ~6k-token CONTEXT.md and skips the research entirely. Each saves ~274k tokens (survey-grade) up to a full 0.6–1.5M (per-paper extraction).

Break-even ≈ 3–7 consumers — or just 1–2 if each would otherwise do full extraction. Past that, every additional loader is near-pure profit. It's per-topic, not per-question: one package serves unlimited related tasks until the field moves.

The asymmetric part — risk avoided. Re-researching doesn't just re-spend compute; it grounds work in undetectable fabrications (our unaided agent cited a real arXiv id with the wrong paper's title — you can't catch that by eye). One fake source in a downstream paper, grant, or memo can invalidate the whole artifact: bounded one-time build cost vs. unbounded downside.

The decision rule

When to run it — and it's self-enforcing

✓ Run kp-build

  • The topic is model-weak — recent, niche, or post-training-cutoff.
  • ≥ 3 agents will reuse the package (or even 1–2 if each would otherwise do full extraction).
  • Downstream work needs zero fabricated sources (papers, grants, strategy memos).

✗ Skip it

  • The model already knows the field — value is structurally ~0 (3 topics tested tied).
  • It's a one-off question with no reuse.
  • You only need traceability, not accuracy — cheaper ways exist.
The falsification gate makes this rule verifiable, not faith-based: it tells you honestly, for free, whether the package beat the unaided baseline — so a package that adds nothing is caught before you trust it.
The one-liner for the board: the engine and verification are free and trustworthy; the cost and the risk both live entirely in the LLM agent work — so the highest-leverage next build is a cheap gate that stops us paying for packages that won't help.
