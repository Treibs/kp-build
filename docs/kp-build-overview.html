kp-build

An open-source tool that builds verified, citation-checked knowledge packages an AI agent can load — so it answers from real, checked sources instead of guessing.

open-source · MIT pip install kp-build 0 hallucinated citations — by construction $0 — verification is free public APIs

Ask an AI assistant about something that matters — a health question, recent research, a decision — and it sounds completely confident. Sometimes it's completely wrong. It will even cite a study that doesn't exist.

kp-build fixes that for a given topic. It produces a knowledge package: a small, self-contained folder of the real literature on a narrow subject —

  • a verified citation spine — every paper checked live against arXiv / Crossref / OpenAlex; the ones that don't check out are dropped, not kept;
  • claims — the key findings, each tied to a quoted line from its source;
  • the open questions the papers themselves flag as unsolved;
  • a debate map — the contested points, what's settled vs. what's hype;
  • and a small briefing file an agent loads to inherit the whole topic at once.

An agent loads it and works from checked knowledge. It's a reusable, machine-checkable asset — not a one-shot answer.

A worked example: what actually improves sleep?

An everyday question, and a good test — the kind of thing people ask AI all the time, where being wrong matters and the internet is full of hype.

We asked a model to review the sleep evidence and cite studies. It cited 10.5665/sleep.6072 — confidently. That DOI resolves to nothing; it was invented. It also engaged only about a quarter of the real evidence.

So we built a verified package on it. Every claim is backed by a real, checked study, and the package's core is a five-way evidence-vs-hype map:

  • CBT-I (behavioral therapy) is the first-line, best-evidence treatment.
  • Low-dose melatonin, timed right, has a small real effect; higher doses don't help more — and most supplements are mislabeled.
  • Mouth-taping, blue-light-blocking glasses, and tracker-driven "optimization" are mostly hype — with the studies behind each call (sleep trackers can even feed insomnia, an effect researchers named "orthosomnia").

Does the package help? The tool measures it: a model with the package vs. one without, on a held-out question. Unaided: f1 0.40 (1 fabricated citation, 6 of 23 studies covered). With the package: 0 fabrications, 17 of 23, f1 0.85.

The package is public — any agent can install it: kpm add github:Treibs/kp-sleep-insomnia-evidence#v0.1.0

How it works

A research skill (you + AI subagents) gathers and drafts; a deterministic engine verifies, assembles, and scores. The engine never trusts the drafting — it checks everything.

$ kp-build build -i sleep.research.json -o pkg
verifying 23 citation(s) against Crossref / OpenAlex ...
  citations: 23/23 verified     validation: OK
  45 claims · 7 open problems · 5 debates

$ kp-build falsify pkg --base base.txt --kp pkg.txt
  base:  cited a study that doesn't exist · covered 6/23
  pkg:   0 fabrications · covered 17/23
  VERDICT: KP HELPS — f1 0.40 → 0.85

① No fake citations

A paper is kept only when an explicit arXiv id or DOI resolves and its real title matches. A "real id, wrong paper" mislabel is rejected. The gate never rescues a bad id with a fuzzy search.

② Grounded claims

Where the source text is reachable, each claim's quoted passage is machine-confirmed to actually appear in the paper — not just asserted.

③ It checks itself

A probe first asks whether the model is even weak on the topic (skip it if not). After building, falsify measures whether the package actually beats an unaided agent — and says so honestly when it doesn't.

Honest about what it is

  • It pays off where the model is weak — recent, niche, or fast-moving topics. On something the model already knows cold, a package adds traceability and completeness, not accuracy — and the falsify check will tell you that plainly instead of selling a hollow win.
  • Confidence is corpus-relative. A claim is only as good as its sources; the package says so rather than asserting absolute truth.
  • A package is a snapshot. It carries the date it was built; when a field moves, a rebuild is a diff.
  • It's a foundation, not an oracle — the verified state of the literature, with every citation checked to exist.

Use it

# install the engine
$ pip install kp-build

# or hand the repo to Claude Code and ask it to set up the /kp-build skill, then:
/kp-build  the evidence on <your everyday or research topic>

The engine is Python (stdlib-heavy; two small dependencies). Verification uses free public APIs — no keys, no cost. Packages are shared through KPM, an open package manager for knowledge — think npm, but the package is verified knowledge instead of code. Build a package once; anyone's agent can install and load it.

The bet: verified knowledge as shareable packages. Build it once, checked, and nobody's agent has to re-research — or re-hallucinate — the same field again.
