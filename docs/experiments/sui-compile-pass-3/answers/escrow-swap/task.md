Write a module implementing a two-party atomic swap escrow. Party A creates an escrow offering a generic item (any object type with appropriate abilities) and naming a counterparty and the exact object type it wants in return is out of scope — instead: the escrow holds A's item, the counterparty deposits their item, and a `swap` function completes the exchange (each party receives the other's item). Either party can cancel before the swap completes and recover whatever they deposited. The escrow object itself must be cleaned up on both paths.
